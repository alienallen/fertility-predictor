# 产品需求文档 (PRD) - 备孕预测应用 v1.0

## 1. 问题陈述 (Problem Statement)

作为备孕女性，我需要准确预测每月最易受孕的时间，以便在最佳时间安排同房，提高受孕成功率。

目前市场上已有的经期/排卵期 App 存在以下问题：
- 单纯依靠周期法计算，准确度有限
- 缺乏多数据源（体温、试纸）组合预测能力
- 功能冗余，包含不必要的社交/社区功能，使用复杂

## 2. 解决方案 (Solution)

一款专注于备孕的智能预测工具小程序/Web应用：
- 通过基础周期、体温、排卵试纸多数据源组合计算，精准预测易孕日
- 以月历形式直观展示经期、易孕日、排卵日
- 支持可选数据录入（体温、试纸），用户可自主选择数据丰富度
- 提供排卵日、体温测量、试纸检测推送提醒
- 保持简洁纯粹的工具属性，无社交功能

## 4. 实现决策 (Implementation Decisions)

### 4.1 架构决策

**应用形式：**
- 阶段一：Web 应用（响应式网页，支持移动端浏览器）
- 阶段二：微信小程序（复用 Web 版核心逻辑）

**技术栈建议：**
- 前端：Vue 3 + uni-app（兼顾 Web + 微信小程序）
- 后端：Node.js + Express/NestJS（无需数据库）
- 存储：本地 JSON 文件 + AES-256 加密
- 推送：极光推送（国内）+ Firebase（海外）

**数据存储设计：**
- 每个用户一个独立的 JSON 文件，按 user_id 命名
- 文件内容 AES-256 加密存储
- 文件命名规则：`data/{user_id}.enc`（加密）或 `data/{user_id}.json`（开发环境）
- 用户密钥由服务器使用用户专属密钥加密后再存储

### 4.2 核心模块

| 模块 | 职责 | 关键接口/功能 |
|------|------|--------------|
| 用户模块 | 微信登录、手机号注册、账户管理 | login, register, bindPhone, getProfile |
| 经期模块 | 经期记录 CRUD、历史查询 | addPeriod, getPeriods, updatePeriod, deletePeriod |
| 体温模块 | 体温记录 CRUD、曲线生成 | addTemp, getTemps, getTempChart |
| 试纸模块 | 试纸结果录入、时间线展示 | addOvulationTest, getOvulationTests |
| 预测引擎 | 多数据源综合计算排卵日、易孕日 | predict, calculateFertility |
| 日历模块 | 月历渲染、日期状态标注 | getMonthData, getDayDetail |
| 通知模块 | 推送提醒配置、触发、发送 | setReminder, triggerReminder |
| 数据模块 | 数据导出、清空 | exportData, clearData |

### 4.3 预测算法设计

**输入数据：**
- 经期记录（必需）：月经开始日期、周期长度、经期持续天数
- 体温记录（可选）：每日基础体温数据
- 试纸记录（可选）：试纸检测结果及时间

**计算逻辑：**
```
1. 基础周期法：
   - 排卵日 = 下次月经开始日 - 14天
   - 易孕期 = 排卵日前5天 + 排卵日后1天

2. 体温辅助：
   - 体温下降日 + 1天 ≈ 排卵日
   - 体温升高后持续高温 = 已排卵

3. 试纸辅助：
   - 强阳转弱/排卵试纸显示排卵 = 排卵进行中
   - 试纸结果优先于周期法

4. 综合决策：
   - 有试纸记录 → 以试纸为准
   - 无试纸但有体温 → 以体温+周期综合判断
   - 仅经期 → 以周期法计算
```

### 4.4 数据结构（JSON 文件存储）

每个用户的加密数据文件：`data/{user_id}.enc`

**文件结构：**
```json

    "wechat_openid": "oxxx",
    "phone": "+86xxx",
    "cycle_length": 28,

    "created_at": "2026-01-01T00:00:00Z",
    "updated_at": "2026-01-01T00:00:00Z"
  },
  "periods": [
    {
      "id": "p001",
      "start_date": "2026-01-01",
      "end_date": "2026-01-05",
      "created_at": "2026-01-01T10:00:00Z"
    }
  ],
  "temperatures": [
    {
      "id": "t001",
      "record_date": "2026-01-01",
      "temperature": 36.5,
      "created_at": "2026-01-01T07:00:00Z"
    }
  ],
  "ovulation_tests": [
    {
      "id": "o001",
      "test_date": "2026-01-15T10:00:00Z",
      "result": 3,
      "created_at": "2026-01-15T10:00:00Z"
    }
  ],
  "reminders": [
    {
      "id": "r001",
      "reminder_type": 1,
      "remind_time": "09:00",
      "enabled": true
    }
  ]
}
```

**字段说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| user | object | 用户基本信息 |
| periods | array | 经期记录列表 |
| temperatures | array | 体温记录列表 |
| ovulation_tests | array | 试纸记录列表 |
| reminders | array | 提醒设置列表 |

**加密方案：**
- 算法：AES-256-GCM
- 密钥派生：PBKDF2（用户登录 token + salt）
- 每个文件独立 IV（Initialization Vector）

## 5. 测试决策 (Testing Decisions)

### 5.1 核心测试场景

| 模块 | 测试重点 |
|------|----------|
| 预测引擎 | 周期法准确性、体温拐点识别、试纸结果解析 |
| 经期记录 | 新增、修改、删除、跨月记录 |
| 体温记录 | 连续记录、曲线生成、异常值处理 |
| 试纸记录 | 多种结果录入、时间线展示 |
| 日历展示 | 跨月切换、日期状态标注、受孕指数显示 |
| 推送提醒 | 定时触发、时区处理、重复提醒 |
| 数据同步 | 多设备登录数据一致性 |

### 5.2 测试数据

- 准备 10+ 个不同周期长度（21-35天）的用户数据
- 准备多种体温曲线形态（典型/异常）
- 准备试纸阳性/阴性/强阳等不同结果场景

## 7. 其他说明 (Further Notes)

### 7.1 隐私合规
- 符合《个人信息保护法》要求
- 经期/体温/试纸属于敏感个人健康数据，需单独授权
- 数据加密存储，传输使用 HTTPS

### 7.2 医疗器械提示
- 本产品为健康类工具，非医疗器械
- 预测结果仅供参考，不作为医疗诊断依据
- 必要时建议咨询专业医生

### 7.3 后续扩展思路
- 成功后可延伸至孕期管理、产检提醒
- 积累用户后可对接生殖医学机构
- 海外市场可对接 Apple HealthKit

## 8. UI 设计规范

**色彩方案：**
| 用途 | 颜色 | 色值 |

| 辅助色（terracotta） | 赤陶色 | #D4A574 |
| 强调色（blush） | 玫瑰粉 | #E8B4B8 |
| 背景 | 奶油白 | #FDF8F3 |
| 文字 | 深暖灰 | #4A4541 |

**日历状态色：**
| 状态 | 颜色 | 用途 |
|------|------|------|

**设计特点：**

- 浮动按钮：展开式菜单
- 配色方案：避免紫蓝渐变，使用自然色系
